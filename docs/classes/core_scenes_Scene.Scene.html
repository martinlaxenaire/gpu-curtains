Class Scene

Used to by the Renderer render everything that needs to be rendered (compute passes and meshes) in the right order with the right pass descriptors and target textures, perform textures copy at the right time, etc.

Render order

  • Run all the ComputePass first, sorted by their renderOrder
  • Then render all renderPassEntries pingPong entries Meshes, sorted by their renderOrder
  • Then all Meshes that need to be rendered into specific renderPassEntries outputTarget entries:
    • First, the opaque unprojected Meshes (i.e. opaque FullscreenPlane, if any), sorted by their renderOrder
    • Then, the transparent unprojected Meshes (i.e. transparent FullscreenPlane, if any), sorted by their renderOrder
    • Then, the opaque projected Meshes (i.e. opaque Mesh, DOMMesh or Plane), sorted by their renderOrder
    • Finally, the transparent projected Meshes (i.e. transparent Mesh, DOMMesh or Plane), sorted by their Z position and then their renderOrder
  • Finally all Meshes that need to be rendered directly to the renderPassEntries screen (the Renderer current texture), in the same order than above.

renderPassEntries

renderPassEntries: RenderPassEntries

A RenderPassEntries object that will contain every Meshes that need to be drawn, put inside each one of our three entries type arrays: 'pingPong', 'outputTarget' and 'screen'.

  • The Scene will first render all renderPassEntries pingPong entries Meshes
  • Then all Meshes that need to be rendered into specific renderPassEntries outputTarget entries
  • Finally all Meshes that need to be rendered to the renderPassEntries screen

renderSinglePassEntry

  • renderSinglePassEntry(commandEncoder, renderPassEntry): void

  • Here we render a RenderPassEntry:

    • Set its renderPass descriptor view or resolveTarget and get it at as swap chain texture
    • Execute onBeforeRenderPass callback if specified
    • Begin the GPURenderPassEncoder | GPU render pass encoder using our renderPass descriptor
    • Render the single element if specified or the render pass entry Stack: draw unprojected opaque / transparent meshes first, then set the camera bind group and draw projected opaque / transparent meshes
    • End the GPURenderPassEncoder | GPU render pass encoder
    • Execute onAfterRenderPass callback if specified
    • Reset pipeline manager current pipeline

    Parameters

    • commandEncoder: GPUCommandEncoder

      current GPUCommandEncoder

    • renderPassEntry: RenderPassEntry

      RenderPassEntry to render

    Returns void

    • Defined in src/core/scenes/Scene.ts:488
