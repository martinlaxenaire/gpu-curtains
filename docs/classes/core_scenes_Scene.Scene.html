Class Scene

Used to by the Renderer to render everything that needs to be rendered (compute passes, meshes and/or render bundles) in the right order with the right pass descriptors and target textures, perform textures copy at the right time, etc.

Render order

  • Run all the ComputePass first, sorted by their renderOrder
  • Then render all renderPassEntries pingPong entries Meshes or RenderBundle, sorted by their renderOrder.
  • Then all Meshes that need to be rendered into specific renderPassEntries outputTarget entries:
    • First, the opaque unprojected Meshes (i.e. opaque FullscreenPlane or RenderBundle, if any), sorted by their renderOrder.
    • Then, the transparent unprojected Meshes (i.e. transparent FullscreenPlane or RenderBundle, if any), sorted by their renderOrder.
    • Then render all renderPassEntries prePass entries ShaderPass, sorted by their renderOrder. This would be mainly used for "blit" passes rendered before the content that needs to be actually rendered on the main screen buffer.
    • Then, the opaque projected Meshes (i.e. opaque Mesh, DOMMesh, Plane) or RenderBundle, sorted by their renderOrder.
    • Finally, the transparent projected Meshes (i.e. transparent Mesh, DOMMesh, Plane or RenderBundle), sorted by their Z position and then their renderOrder.
  • Next, all the Meshes that need to be rendered directly to the renderPassEntries screen main buffer (the Renderer current texture), in the same order than above. The screen renderPassEntries array can have multiple entries, allowing you to render first a given set of meshes, the perform any operation (like getting the outputted content) before rendering another set of meshes.
  • Finally, all the post processing ShaderPass in the postProPass renderPassEntries array.

renderPassEntries

renderPassEntries: RenderPassEntries

A RenderPassEntries object that will contain every Meshes or RenderBundle that need to be drawn, put inside each one of our three entries type arrays: pingPong, renderTarget and screen.

  • The Scene will first render all renderPassEntries pingPong entries Meshes.
  • Then all Meshes that need to be rendered into specific renderPassEntries renderTarget entries.
  • Finally all Meshes that need to be rendered to the renderPassEntries screen.

renderSinglePassEntry

  • renderSinglePassEntry(
        commandEncoder: GPUCommandEncoder,
        renderPassEntry: RenderPassEntry,
    ): void

  • Here we render a RenderPassEntry:

    • Set its renderPass descriptor view or resolveTarget and get it at as swap chain texture
    • Execute onBeforeRenderPass callback if specified
    • Begin the GPU render pass encoder using our renderPass descriptor
    • Render the single element if specified or the render pass entry Stack: draw unprojected opaque / transparent meshes first, then set the camera and lights bind group and draw projected opaque / transparent meshes
    • End the GPU render pass encoder
    • Execute onAfterRenderPass callback if specified
    • Reset pipeline manager current pipeline

    Parameters

    • commandEncoder: GPUCommandEncoder

      current GPUCommandEncoder

    • renderPassEntry: RenderPassEntry

      RenderPassEntry to render

    Returns void
